    JAVA高并发秒杀系统项目

    1 项目框架搭建

    1.1 SpringBoot环境搭建

    使用IDEA编辑器，创建JAVA项目。项目配置如下：

    基础配置：

    • Project： Maven
    • language: java
    • Spring Boot

    project Metadata:

    • Group: com.xxxx
    • Artifact: seckill
    • Package: com.xxxx.seckill

    Dependencies:

    • Spring Web: 构建Web接口
    • MyBatis-plus Framework: 数据库ORM
    • Lombok：简化代码
    • Redis：处理缓存
    • MySQL Driver：数据库连接
    • RabbitMQ：消息队列

    配置application.yml:

    创建启动类 SeckillApplication.java。

    配置MyBatis Mapper扫描路径，即在启动类中添加@MapperScan注解

    创建Mapper接口和xml文件

    XML文件需要放在src/main/resources/mapper目录下

    启动项目，判断项目是否运行成功。

    注意事项：

    确保本地已安装MySQL和Redis，并创建Seckill数据库。

    1.2 集成Thymeleaf，respBean

    集成Thymeleaf模版引擎：用于服务端渲染HTML页面。

    • 在pom.xml中配置Thymeleaf的依赖

    • 在application.yml文件中配置Thymeleaf

    • 创建html模版，在src/main/resources/templates 下创建页面，如item.html

    • 在Controller中返回页面

    集成RespBean统一响应体：标准化API响应格式，方便前端处理

    • 在vo文件夹，创建RespBean类
    • 在Controller中返回RespBean

    结合Thymeleaf和RespBean ：在页面中通过AJAX调用接口并使用RespBean响应

    静态页面配置：将CSS/JS文件放在src/main/resources/static目录下，Thymeleaf会自动映射

    1.3 MyBatis-plus

    MyBatis-plus基础配置

    • 添加依赖：在pom.xml中添加MyBatis-plus依赖
    • 配置数据源和MyBatis-plus：在application.yml中配置数据库源，和MyBatis-plus的配置

    实体类与Mapper接口

    • 创建实体类：在src/main/java/com/xxxx/seckill/pojo文件夹中创建Goods.java文件，添加@Data注解
    • 定义Mapper接口：在`src/main/java/com/xxxx/seckill/Mapper文件夹中创建GoodsMapper.java文件，添加@Mapper注解
    • 创建xml映射文件：在src/main/resources/mapper/中创建GoodsMapper.xml文件

    Service层使用MyBatis-plus

    • Service接口继承IService，即在IGoodsService.java文件中，实现IGoodsService 继承IService
    • Service实现类继承ServiceImpl 同理相似处理，即GoodsServiceImpl 继承 ServiceImpl

    2 分布式会话

    2.1 用户登录

    2.1.1设计数据库

    设计用户表，在MySQL中进行设计，设计了五个表。t_user, t_goods, t_order, t_seckill_goods, t_seckill_order, 以及每个 表中具体的参数如下：

    • t_user表中包含的字段
    • id: 用户ID
    • nickname：用户名称
    • password：用户密码
    • salt：盐值用于加密
    • head：头像
    • register_date: 注册时间
    • last_login_date : 最后一次登录时间

    • login_count: 登录次数

    • t_goods表中包含的字段

    • id: 商品ID
    • goods_name: 商品名称
    • goods_title: 商品标题
    • goods_img: 商品图片
    • goods_detail: 商品详情
    • goods_price: 商品价格
    • goods_stock: 商品库存
    • t_order表中包含的字段:
    • id: 订单ID
    • user_id: 用户ID
    • goods_id: 商品ID
    • delivery_addr_id: 收货地址ID
    • goods_name: 商品名称
    • goods_count: 商品数量
    • goods_price: 商品单价
    • goods_channel: 订单渠道
    • status: 订单状态
    • create_date: 订单创建时间
    • pay_date: 支付时间
    • t_seckill_goods表中包含的字段：
    • id: 订单ID
    • goods_id: 商品ID
    • seckill_price: 秒杀价格
    • stock_count: 库存数量
    • start_data: 秒杀开始时间
    • end_data: 秒杀结束时间
    • t_seckill_order表中包含的字段：
    • id: 秒杀订单ID
    • user_id :用户ID
    • order_id : 订单ID
    • goods_id : 商品ID

    2.1.2明文密码二次MD5加密

    防止明文传输的截获，避免数据库泄露后密码被反推。

    编写MD5Util.java工具类

    • 创建utils文件夹，添加MD5Util.java工具类
    • 设置两个加密函数，前端一次，后端一次

    使用工具类，在Service中使用，即UserServiceImpl.java文件中，判断用户输入密码和数据库中中存储的密码是否一致

    2.1.3 参数校验+全局异常处理

    作用：参数校验，对输入的参数进行校验，如用户名和密码等

    • 创建ValidatorUtil.java工具类
    • isMobile.java验证文件，验证密码和手机号

    统一处理参数校验异常，业务异常等

    • 创建exception文件夹，添加GlobalExceptionHandler.java和GlobalException.java文件

    2.2 共享Session

    2.2.1 SpringSession集成

    实现分布式会话管理。作用：将Session存储从Tomcat默认内存迁移到Redis中，实现多服务实例间的Session共享。

    • 添加依赖：在pom.xml中添加Spring Session + Redis依赖
    • 配置Spring Session：在application.yml中配置，如启动Redis作为Session存储
    • 启动Session：在LoginController.java中添加参数，在IuserService.java中添加方法，在UserServiceImpl.java中具体实现方法，将用户信息存入到redis中

    2.2.2 Redis

    Redis配置优化

    • 在config文件夹下，创建RedisConfig.java文件
    • 配置序列化文件，令其支持JSON文件
    • 检查Redis中是否有Session键

    3.3 秒杀

    秒杀核心逻辑：

    • 用户提交秒杀请求
    • Redis预减库存，原子操作
    • 库存不足---返回库存不足---提示失败
    • 库存充足---检查用户是否重复秒杀
    • 重复请求---返回重复提示
    • 首次请求---
      • Server对RabbitMQ发送秒杀信息---
      • RabbitMQ向Server表示确认接受---
      • Server对用户：返回抢购中，异步处理---
      • RabbitMQ对数据库：扣减库存+生成订单---
      • 数据库对RabbitMQ：返回操作结果

    后端实现：

    • 消息队列消费者，进行订单处理：SeckillController.java。功能，预减库存，生成订单，更新Redis中的用户购买记录

    6 接口优化

    接口优化可以提高系统吞吐量和稳定性。

    6.1 Redis预减库存减少数据库的访问

    通过Redis原子操作拦截无效请求，避免高频访问数据库。可以拦截90%以上的无效请求，原子操作可保证线程安全。

    6.2 内存标记减少Redis的访问

    在服务器本地内存中维护商品的状态，减少Redis的访问频率。原因本地内存访问速度更快。

    7 安全优化

    7.1 秒杀接口地址隐藏

    目的：防止攻击者通过抓包直接获取接口URL进行恶意请求。

    实现方案：

    • 动态生成接口路径：在秒杀开始前，前端通过独立接口获取动态加密的秒杀地址
    • 接口逻辑验证：验证路径是否合法

    优势：URL动态变化，每次秒杀的请求路径是唯一的，避免脚本攻击。

    7.2算术验证码

    通过人机验证拦截机器人刷单。

    • 生成验证码。
    • 接口继承验证码：前端展示，后端验证。
    • 将验证码结果缓存到Redis中，有效期60秒，避免重复计算

    7.3 接口防刷

    目的：防止用户高频调用接口，保护系统资源。

    方法：使用计数器算法。如每秒内，最多点击5次。

    8 主流的秒杀方案总结

    分层过滤架构：通过逐层过滤无效请求，减少对核心逻辑的冲击。

    分层模型：

    • CDN静态资源缓存：商品详情页面静态化（HTML+CSS+JS），通过CDN分发，减少后端服务器压力。如使用Nginx配置静态资源缓冲策略。

    • 网关层限流：基于IP、用户ID、设备指纹等维度限制请求频率。

    • 服务层异步优化：请求进入消息队列，异步处理订单：削峰填谷，避免服务雪崩。

    流量削峰方案：将瞬时流量转化为平稳流量，保护系统稳定性。

    库存扣减方案：避免超卖，保证数据一致性。

    • 预扣库存
    • Redis原子操作
    • 数据库行锁
