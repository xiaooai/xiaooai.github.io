    github page创建方法

    1 创建新的github仓库

    • Repository name 中的“username”修改为自己的用户名，如在我的网站上修改为“xiaooai”
    • 其他设置为默认即可

    2 MKdocs说明

    MKdocs可以处理markdown文件，并方便上传到github page中。优点：

    • 处理markdown文件
    • 多种渲染主题，利用material工具
    • 可以本地预览

    安装步骤如下：

    • 1 使用pip下载Mkdocs库
    pip install mkdocs
    • 2 在本地创建文件夹
    mkdocs new my-project
cd my-project
    • 3 上面的命令运行之后，会出现一些文件，说明如下
    文件名 类型 作用
    mkdocs.yml YML 处理网站中显示的结构框架，如导航栏等
    docs 文件夹 存储网站中显示的内容
    docs/index.md markdown 存储书写内容的文件
    • 4 本地运行，进行网站文件预览
    mkdocs serve

    打开http://127.0.0.1:8000/在浏览器中，可以显示出目前建立的主页

    3 github部署站点

    看了网络上的其他教程，认为最方便的是将本地的文件进行处理后，得到可以直接上传到部署网站的文件，更为方便。步骤如下：

    • 1 进入my-project文件夹，运行下列命令，生成站点文件site
    mkdocs build
    • 2 在本地合适位置将之前创建的仓库进行clone
    git clone git@github.com:username/username.github.io.git
    • 3 将site文件夹中的内容，全部粘贴到clone的仓库中
    • 4 在本地仓库，进行GitHub的上传
    git add .
git commit -m "update"
git push -u origin master
    • 5 访问网站

    现在就可以访问自己的网站了，网站名为https://username.github.io，将username替换成自己的名字，例如本例中为xiooai。

    4 进一步美化

    利用material工具进行美化网站界面，仅需修改mkdocs.yml文件即可，步骤如下：

    • 1 安装必要的material库
    pip install mkdocs-material mkdocs mkdocs-roamlinks-plugin mkdocs-exclude
    • 2 修改mkdocs.yml文件，在文件中添加以下代码，并保存
    theme: 
  name: material
  language: zh
  features:
    - navigation.expand
    - tabs
    • 3 与上面步骤相同，进行本地预览。
    mkdocs serve
    • 4 重复GitHub部署站点的操作，即可看到网站进行更新。

    注：网站部署需要一段时间，需要稍微等几分钟即可
